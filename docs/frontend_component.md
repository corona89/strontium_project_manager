# Frontend Design: Trello Copy 🎨

## 1. 컴포넌트 구조
- **Board Container**: 사이드바 및 메인 캔버스 관리
- **List Component**: 카테고리 제목 및 카드 목록 컨테이너
- **Card Component**: 요약 정보 표시 및 드래그 핸들
- **Detail Modal**: 마크다운 편집기, 댓글 창, 체크리스트 포함

## 2. UI 핸들링 전략 (낙관적 UI)
1. 유저가 카드를 옮기면 **DOM 위치를 즉시 변경**하고 서버에 `PUT` 요청을 보낸다.
2. 서버 응답이 실패(4xx, 5xx)하면 **원래 위치로 카드를 롤백**하고 경고창을 띄운다.
3. 다른 사용자에 의한 변경(`WebSocket`)은 해당 카드의 ID를 찾아 **해당 엘리먼트만 갱신**하여 불필요한 전체 렌더링을 방지한다.

## 3. 권한 로직
- 모든 API 호출 전 `X-Board-ID` 헤더를 포함하며, 서버 서비스 계층에서 요청 유저가 해당 보드 멤버인지 반드시 SQL Join을 통해 검증한다.
